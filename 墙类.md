# 墙类 (Walls)

墙体是 TPT 中的静态阻挡结构，用于构建容器、管道、电路隔离等。不同类型的墙体对粒子、压力、能量有不同的通透性。墙类共 **19** 种，其中 3 种为工具型（清除墙、清除一切、风向计），其余 16 种为功能型墙体。

## 清除墙 (Erase)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_ERASE` |
| 颜色 | 无（工具型，不渲染颜色） |
| 通透性 | N/A（工具而非墙体） |
| 导电性 | 否 |
| 特殊属性 | 橡皮擦，移除被点击位置的所有墙类 |

### 核心行为

清除墙是一个纯粹的编辑工具，不会在模拟中创建任何持久结构。它的唯一功能是移除已经放置的墙体。

- **擦除模式**：使用后，点击或拖拽经过的所有墙体格子将被移除，恢复为空白。
- **作用范围**：仅作用于墙体（Walls），不会清除该位置的粒子或标记（Signs）。
- **不影响模拟**：作为工具，它在放置后不留下任何墙体实体，因此不会对粒子的运动、压力传导、电路信号产生任何影响。

### 与清除一切 (Eraser) 的区别

| 特性 | 清除墙 (Erase) | 清除一切 (Eraser) |
|------|---------------|-------------------|
| 删除墙 | 是 | 是 |
| 删除粒子 | 否 | 是 |
| 删除标记 | 否 | 是 |

### 应用场景

- 修改已建成的墙体结构中的错误
- 拆除临时隔离墙
- 为需要粒子通过的区域打开缺口

## 导体墙 (ConductiveWall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_CNDTW` |
| 颜色 | 浅灰色 |
| 通透性 | 完全阻挡（所有粒子 + 压力均无法通过） |
| 导电性 | **是**（可在墙体之间传导 SPRK 电脉冲） |
| 特殊属性 | 兼具墙的完全阻挡能力与导体的导电能力 |

### 核心行为

导体墙是基础墙（Wall）的导电升级版。它在物理阻挡方面与基础墙完全相同——所有类型的粒子（粉末、液体、气体、固体、能量粒子）以及气压均被完全阻挡。区别在于，导体墙可以像金属导线一样传导 SPRK（电火花）信号。

- **阻挡范围**：粉末（SAND, DUST, STNE 等）、液体（WATR, OIL, LAVA 等）、气体（GAS, SMKE, O2 等）、能量（PHOT, NEUT, ELEC 等）、固体（ICE, WOOD, GLAS 等）——全部无法穿过。
- **导电行为**：当相邻的导体墙、导体（Conductor）或其他导电元素接收到 SPRK 信号时，导体墙会将信号传递到所有相邻的导电墙体，形成连锁反应。SPRK 信号在导体墙之间的传播速度与普通金属导体一致。
- **压力阻挡**：气压（Air Pressure）同样被完全阻挡，两端压力完全隔离。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SPRK | 可沿导体墙传导，加热被接触的粒子 |
| WATR/OIL/LAVA | 被完全阻挡，堆积在墙边 |
| PHOT（光子） | 被阻挡，光子在墙面上反射或消失 |
| NEUT（中子） | 被阻挡 |
| SAND/DUST | 堆积在墙面上方 |
| 气压 | 被完全隔离，墙两侧压力互不影响 |
| PSCN/NSCN | SPRK 信号可通过这些半导体传入/传出导体墙网络 |
| INST | 导体墙可被 INST（绝缘体）隔断，从而控制电路走向 |

### 应用场景与构建示例

**场景 1：导电围墙**
在反应器周围用导体墙建造围墙，既阻止内容物外泄，又可作为电路的一部分传输信号。例如：用导体墙围住一片区域，在墙外用 PSCN 和 NSCN 搭建控制电路，通过墙本身传递开关信号。

**场景 2：墙体电路 — 无导线信号传输**
在大型装置中，如果空间有限无法放置独立导线（METL），直接用导体墙作为"墙体总线"传递信号。例如：在多层建筑中，每层之间的信号通过承重墙（导体墙）垂直传递。

**场景 3：与 INST 配合的定向电路**
```
导体墙 → INST（断点）→ PSCN → 目标元件
```
利用 INST 的绝缘特性在导体墙上制造"断点"，将信号从墙体引出到 PSCN，再通过 NSCN 继续传播，实现从墙体到元件的信号跳线。

**场景 4：电击陷阱**
用导体墙围成狭小通道，在其中填充 WATR 或 GAS。当 SPRK 信号沿导体墙传导时，墙体会加热相邻粒子（导体墙在传导 SPRK 时温度升高），从而实现对大范围区域的加热或点燃效果。

## 电控墙 (E-Wall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_EWALL` |
| 颜色 | 绿色（未通电时较暗，通电时变亮） |
| 通透性 | 未通电时完全阻挡；通电时完全通透 |
| 导电性 | **是**（接收 SPRK 信号来控制开关状态） |
| 特殊属性 | 电控开关门 — 通电开启，断电关闭 |

### 核心行为

电控墙是一种状态切换型墙体。它的通透性完全由是否接收到 SPRK 信号决定。

- **未通电状态（默认）**：行为等同于基础墙（Wall）。阻挡所有粒子类型（粉末、液体、气体、固体、能量）和气压。粒子在墙边堆积，无法通过。
- **通电状态**：当 E-Wall 接收到 SPRK 信号后，该墙体格子变为完全透明。所有粒子（任何类型）和气压均可自由穿过，仿佛墙体不存在。
- **信号维持**：只要持续接收到 SPRK 信号，E-Wall 就保持开启状态。一旦断电，立即恢复为阻挡状态，此时恰好位于墙内的粒子将被困住或被推出。
- **逐格独立**：每个 E-Wall 格子是独立工作的，仅接收 SPRK 的格子会变为透明。可以用部分通电的方式只激活一面墙的某些部分。

### 具体交互示例

| 粒子/元素 | 未通电 | 通电 |
|-----------|--------|------|
| SAND/DUST | 阻挡 | 自由穿过 |
| WATR/OIL | 阻挡 | 自由穿过 |
| GAS/SMKE | 阻挡 | 自由穿过 |
| PHOT/NEUT | 阻挡 | 自由穿过 |
| SPRK | 接收到则通电开启 | N/A |
| 气压 | 阻挡 | 穿透 |
| INST | E-Wall 可接收 INST 传递的 SPRK（通过接触） | N/A |

### 应用场景与构建示例

**场景 1：定时释放门**
```
[时钟电路] → SPRK 脉冲 → [E-Wall 门] → [粒子储存区]
```
使用定时电路每隔一定帧数向 E-Wall 发送一次脉冲，实现周期性释放粒子。可用于滴液装置、定时投料器等。

**场景 2：粒子气闸 (Airlock)**
```
[外层 E-Wall 门]  [中层空间]  [内层 E-Wall 门]
```
先开外层门放入粒子，关外层门，再开内层门释放粒子。两个 E-Wall 交替通断，实现粒子的"无泄漏传输"，类似太空站气闸原理。通过简单的双稳态电路控制两个门的交替开关。

**场景 3：逻辑控制通道**
利用 E-Wall 构建 AND/OR 逻辑：
- **OR 逻辑**：多个 PSCN 从不同方向连接到同一 E-Wall，任一信号到达即开门。
- **AND 逻辑**：将两个 E-Wall 串联在同一通道中，两者都通电时通道才通畅。

**场景 4：安全泄压阀**
在封闭反应器顶部安装 E-Wall，连接到压力传感器（检测器）。当内部压力过高导致粒子堆积触发检测器时，E-Wall 自动打开释放压力，压力恢复正常后自动关闭。

## 检测器 (Detector)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_DTECT` |
| 颜色 | 蓝灰色 |
| 通透性 | 完全阻挡（所有粒子 + 压力均无法通过） |
| 导电性 | **是**（作为信号输出端，检测到粒子时发出 SPRK） |
| 特殊属性 | 粒子存在性传感器 — 任何粒子进入检测器格子时输出 SPRK |

### 核心行为

检测器是一个传感器型墙体。它检测自身所在格子是否存在任何粒子，并将检测结果转换为 SPRK 电信号输出。

- **检测机制**：逐帧检测。只要该格子内有任何类型的粒子（粉末、液体、气体、能量、固体），检测器就输出 SPRK 信号。
- **输出方式**：SPRK 信号从检测器向所有相邻的导电墙体（导体墙、导体、其他检测器等）传播。相邻的 PSCN/NSCN 等半导体也可以接收此信号。
- **持续输出**：只要粒子持续存在于该格子，检测器就持续输出 SPRK。粒子离开后，信号立即停止。
- **阻挡行为**：与基础墙相同，完全阻挡所有粒子和气压。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| 任何粒子（SAND/WATR/GAS/PHOT 等） | 触发检测器输出 SPRK |
| 空格子 | 检测器无输出 |
| SPRK | 可从其他源传入检测器，但检测器本身的输出是单向的（仅作为信号源） |
| 气压 | 被完全阻挡，气压不触发检测器 |
| METL/PSCN/NSCN | 可接收检测器输出的 SPRK 信号 |
| 相邻的导体墙/导体 | 接收 SPRK 并继续传播 |

### 应用场景与构建示例

**场景 1：液位传感器**
在容器壁的中部或底部嵌入检测器。当液体（如 WATR）填充到检测器所在高度时，检测器输出信号，触发关闭进水阀（E-Wall）或启动排水泵（风扇）。这是自动化液位控制的基础。

**场景 2：粒子计数器/触发器**
```
[管道] → [检测器] → [计数器电路] → [执行器]
```
粒子经过检测器时产生脉冲，通过锁存器或计数器电路统计脉冲数量。当计数达到阈值时触发后续动作。适用于投料计数、精确剂量控制。

**场景 3：安全警报系统**
在反应堆外壁内侧嵌入检测器阵列。正常情况下墙内无粒子（都被反应器内部容纳），但一旦出现泄漏，粒子接触到检测器立即触发警报信号，切断电源或关闭其他通道。

**场景 4：自动门 — 接近开关**
```
[检测器] → [NOT 门] → [E-Wall 门]
```
无人/无粒子时 E-Wall 保持关闭；当粒子接近并触碰检测器时，E-Wall 打开让粒子通过。常用于自动分拣系统中的通道控制。

## 风向计 (Streamline)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_STRM` |
| 颜色 | 白色/半透明（仅在 Streamline 显示模式下可见） |
| 通透性 | N/A（工具型，不创建实体墙体） |
| 导电性 | 否 |
| 特殊属性 | 可视化工具 — 显示气压流动方向和强度，不参与物理模拟 |

### 核心行为

风向计是纯粹的调试和可视化工具，不对模拟产生任何影响。

- **放置方式**：将风向计放置在一个格子上，系统将从该点开始追踪气压梯度，绘制一条或多条气流线（streamline）。
- **显示内容**：气流线的密度和方向反映了该区域的气压分布和风速。线越长、越多，表示气流越强。
- **零影响**：不会阻挡任何粒子、不会影响气压分布、不会参与电路。即使放置在粒子密集区域，模拟结果与未放置风向计时完全一致。
- **显示模式**：需要在 TPT 的显示设置中开启 "Streamline" 或 "Air" 视图才能看到流线效果。

### 应用场景

- 分析风扇布局的气流覆盖范围，优化风扇位置和角度
- 调试压力装置（如用风扇和压力墙构建的气压系统），检查是否形成了预期的气流回路
- 研究封闭容器中气压平衡过程
- 教学演示——直观展示 TPT 的气压模拟机制

## 风扇 (Fan)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_FAN` |
| 颜色 | 蓝色（深浅表示强度，箭头方向表示风向） |
| 通透性 | N/A（主动元件，产生气压而非阻挡/通过） |
| 导电性 | 否 |
| 特殊属性 | 使用直线工具拖拽设置方向和强度；拖拽越长，风力越强 |

### 核心行为

风扇是 TPT 中唯一的主动气压源。它不阻挡任何粒子，而是在每一帧向指定方向施加气压推动。

- **方向与强度**：放置风扇时，必须使用直线工具（Line Tool）拖拽。拖拽的起点和终点决定风扇的方向向量。拖拽长度与风扇输出功率成正比——拖得越长，产生的风速越强。
- **作用机制**：风扇修改其前方格子的空气速度（air velocity），从而推动轻质粒子。风扇不直接作用于粒子，而是通过气压间接影响。
- **对粒子的影响**：轻粒子（气体如 GAS、SMKE，以及细小粉末如 DUST）受风扇影响显著；重粒子（如 LAVA、STNE）受风扇影响较小。液体粒子受中等程度影响。
- **不参与电路**：风扇本身不可导电，也不受 SPRK 控制。它始终持续工作。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| GAS/SMKE/O2/H2 | 被显著吹动，沿风向快速移动 |
| WTRV（水蒸气） | 被显著吹动 |
| DUST | 被中等程度吹动 |
| WATR/OIL | 受中等影响，可被吹动但需要较强风力 |
| SAND | 需较强风力才能移动 |
| STNE/LAVA | 几乎不受影响（太重） |
| PHOT/NEUT | 不受气压影响，风扇对其无效 |
| SPRK | 无交互，风扇不受电控 |
| 气压 | 风扇主动产生气压，是气压的源头而非受体 |

### 应用场景与构建示例

**场景 1：粒子分拣 — 风力分离器**
```
[混合粒子入口]
    ↓
[风扇 → 水平吹风]
    ↓         ↓
  [轻粒子]  [重粒子]
  (GAS/SMKE) (SAND/STNE)
```
将混合粒子从上方倒入水平风道。风扇产生横风，轻粒子被吹到远处收集区，重粒子在近处落下。通过调节风扇强度可以调节分拣阈值。

**场景 2：持续气流循环系统**
```
[风扇 →] →→→ [气体墙通道] →→→
   ↑                        ↓
   ←←← [回流通道] ←←← [U 形弯]
```
用气体墙构建封闭的气流循环管道，风扇提供持续动力。适用于气体传输、温度传递（热气体携带热量循环）等应用。

**场景 3：蒸发加速器**
在水面上方放置向下吹的风扇，加速 WATR 蒸发成 WTRV（水蒸气）。风带走水面上的湿气，促进更多蒸发。

**场景 4：温度扩散控制**
将加热元件（如 HEAT/PLSM/热金属）置于风扇上风口，风扇将热空气吹向目标区域，实现定向加热。同样可用于冷却（用冷气体）。

## 液体墙 (LiquidPermeable)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_LIQD` |
| 颜色 | 浅蓝色 |
| 通透性 | 选择性通透 — 仅液体可通过 |
| 导电性 | **是**（可在墙体之间传导 SPRK 信号） |
| 特殊属性 | 液体选择性过滤器，兼具导电能力 |

### 核心行为

液体墙是一种选择性过滤墙体。它在阻挡大多数物质的同时，允许液体类型粒子自由穿过。同时它具备导电能力，可作为"湿式电路"的一部分。

- **允许通过的粒子（TYPE_LIQUID）**：
  - WATR（水）
  - OIL（油）
  - LAVA（岩浆）
  - ACID（酸）
  - SLTW（盐水）
  - MWAX（熔融蜡）
  - 以及其他所有液体类型元素

- **阻挡的粒子**：
  - 粉末类：SAND, DUST, STNE, CNCT, SALT, BRMT, BREL 等
  - 气体类：GAS, SMKE, O2, H2, WTRV, PLSM 等
  - 固体类：ICE, WOOD, GLAS, IRON 等
  - 能量类：PHOT, NEUT, ELEC, PROT 等

- **压力行为**：气压被完全阻挡。
- **导电行为**：同导体墙，SPRK 可在液体墙之间传播。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| WATR | 自由穿过，不受阻挡 |
| OIL | 自由穿过，不受阻挡 |
| LAVA | 自由穿过（注意：LAVA 可能加热墙体） |
| ACID | 自由穿过（注意：ACID 会腐蚀墙体附近的物质） |
| SAND/DUST | 被完全阻挡，堆积在墙面上方 |
| GAS/SMKE | 被完全阻挡 |
| PHOT | 被阻挡 |
| SPRK | 沿墙体传导 |
| WATR + SPRK | 液体墙可在传导 SPRK 的同时让 WATR 穿过，两者互不干扰 |

### 应用场景与构建示例

**场景 1：油水分离器**
```
[SAND + WATR + OIL 混合物]
         ↓
    ═══[液体墙]═══  ← 液体穿过
         ↓
    [液体收集区]    [粉末收集区（留在上方）]
```
将沙、水、油的混合物倒在液体墙上。水、油、岩浆等液体穿过墙体落入下方收集区，沙子和固体粉末留在上方。实现固液分离。

**场景 2：液体导管 — 多相管道**
```
[WATR 源] → ═══[液体墙管道]═══ → [WATR 目标]
               （SAND/DUST 被挡在管道外）
```
用液体墙构建封闭管道，WATR 在管道内自由流动，但外部掉落的 SAND、DUST 等杂物无法进入管道造成堵塞。这比用基础墙建管道更灵活——可以在管道上方撒粉末而不影响管道内液体流动。

**场景 3：导电湿式电路**
利用液体墙的导电性，构建"以液体为介质"的电路。例如：在 WATR 流过的通道两侧放置液体墙作为电极，SPRK 沿液体墙传导，加热流过的水。

**场景 4：岩浆安全通道**
用液体墙构建 LAVA 排放管道。LAVA 自由流动穿过液体墙到达指定排放点，而其他杂物被阻挡，避免管道堵塞和意外反应。

## 吸收墙 (AbsorbingWall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_ABSRB` |
| 颜色 | 暗红色 |
| 通透性 | 对粒子完全吸收（销毁）；对压力完全通透 |
| 导电性 | 否 |
| 特殊属性 | 物质黑洞 — 接触到的任何粒子被立即销毁，压力可自由穿过 |

### 核心行为

吸收墙是一个粒子销毁器。任何接触到吸收墙的粒子都会立即被从模拟中移除，不留痕迹。但与此同时，气压可以自由穿过吸收墙。

- **粒子销毁**：所有类型的粒子（粉末、液体、气体、能量、固体）接触到吸收墙格子时，立即被删除。粒子不会积累、不会反应、不会产生热量。
- **压力通透**：气压完全不受阻碍地穿过吸收墙。吸收墙两侧的气压始终相等（在没有其他干扰源的情况下）。
- **不可导电**：吸收墙本身不参与电路，不能传导 SPRK 信号。
- **逐帧销毁**：粒子在每一帧与吸收墙接触时被销毁。在极高速情况下，粒子可能在进入吸收墙格子后的下一帧才被移除。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SAND/DUST/STNE | 接触到吸收墙即被销毁 |
| WATR/OIL/LAVA | 接触到吸收墙即被销毁 |
| GAS/SMKE/WTRV | 接触到吸收墙即被销毁 |
| PHOT/NEUT/ELEC | 接触到吸收墙即被销毁 |
| SPRK | 不可传导，不影响吸收墙 |
| 气压 | 自由穿过，不受任何阻挡 |
| ACID | 被销毁，不会腐蚀吸收墙 |
| FIRE/PLSM | 被销毁，不会点燃吸收墙 |

### 应用场景与构建示例

**场景 1：溢出清理器**
```
[主反应器] → [溢出通道] → ═══[吸收墙]═══
```
在反应器顶部或侧面安装吸收墙。当反应过于剧烈导致粒子溢出时，溢出的粒子被吸收墙销毁，避免污染外围电路或结构。

**场景 2：压力均衡窗**
```
[高压区] |═══[吸收墙]═══| [低压区]
           （压力自由通过）
```
在需要保持压力均衡但不想有粒子交换的两个区域之间放置吸收墙。任何试图穿过边界的粒子都会被销毁，但压力可以自由平衡——实现"透气不透物"的效果。

**场景 3：无限排水口**
在容器底部放置吸收墙，当液体到达底部时被自动消除。用于模拟"无限排放"场景，如模拟河流下游或开放的排水系统。

**场景 4：净化过滤器 — 多级过滤**
```
[混合物质] → [液体墙] → [气体墙] → [粉末墙] → ═══[吸收墙]═══（最终垃圾桶）
      ↓             ↓           ↓              ↓
   (收集液体)   (收集气体)  (收集粉末)     (无用残余销毁)
```
在分拣系统的末端安装吸收墙，销毁所有未被前面各级过滤器捕获的残余粒子，实现彻底的废物清理。

## 墙 (Wall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_WALL` |
| 颜色 | 深灰色 |
| 通透性 | 完全阻挡 — 无任何粒子或压力可通过 |
| 导电性 | **否** |
| 特殊属性 | 最基础的墙体，无任何特殊能力 |

### 核心行为

基础墙是 TPT 中最简单也是最彻底的阻挡结构。它的行为非常明确：阻挡一切，不通电。

- **粒子阻挡**：所有类型的粒子——粉末（SAND, DUST 等）、液体（WATR, OIL 等）、气体（GAS, SMKE 等）、能量（PHOT, NEUT 等）、固体（ICE, WOOD 等）——均被完全阻挡，无法穿过。
- **压力阻挡**：气压被完全阻挡。墙两侧的压力完全隔离，互不影响。
- **温度影响**：墙体本身不参与热传导（或传导极为缓慢，取决于 TPT 版本的墙体热设置），通常被视为绝热体。
- **不可导电**：SPRK 信号无法穿过基础墙。如果需要导电，应使用导体墙（ConductiveWall）替代。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SAND/DUST | 堆积在墙面和墙上方 |
| WATR/OIL | 被阻挡，形成液面 |
| GAS/SMKE | 被阻挡，在墙边聚集 |
| PHOT | 被阻挡并反射或消失 |
| NEUT | 被阻挡 |
| SPRK | 无法穿过，基础墙是电路中的断点 |
| 气压 | 被完全隔离 |
| HEAT（热量） | 基本不传导（取决于模拟设置） |

### 应用场景与构建示例

**场景 1：反应容器**
这是最基础的应用。用基础墙围成封闭或半封闭的容器，在其中进行化学反应、物理实验。基础墙保证内容物不会泄漏。

**场景 2：电路隔离**
```
[电路 A] |═══[Wall]═══| [电路 B]
```
在两个独立的电路之间放置基础墙，确保电路 A 和电路 B 完全隔离，信号不会意外串扰。使用基础墙而非导体墙，避免不必要的信号传播。

**场景 3：防爆隔离墙**
在爆炸性实验区域外围用基础墙建造双层隔离墙。即使内层被炸毁（如 BOMB 或 FIREWORKS 爆炸），外层基础墙仍能提供第二道防线。

**场景 4：迷宫/障碍物**
利用基础墙搭建物理迷宫，测试粒子的流动行为（如测试液体的路径选择、气体的扩散路径等）。

## 压力墙 (NonpermeableWall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_AIR` |
| 颜色 | 灰蓝色 |
| 通透性 | 选择性通透 — 压力可通过，物质不可通过 |
| 导电性 | 否 |
| 特殊属性 | 透气不透物 — 实现物质隔离的同时保持压力均衡 |

### 核心行为

压力墙的设计理念是"隔离物质，但不隔离压力"。它在阻止所有粒子穿过的同时，允许气压自由扩散。

- **粒子阻挡**：所有类型的粒子（粉末、液体、气体、能量、固体）均被阻挡。
- **压力通透**：气压可以自由穿过。如果墙一侧气压较高，压力会通过压力墙扩散到另一侧，最终两侧压力趋于平衡。
- **不可导电**：不传导 SPRK 信号。
- **与基础墙的对比**：基础墙同时阻挡物质和压力；压力墙仅阻挡物质。在需要在封闭容器中保持与外界的压力平衡时，压力墙是更好的选择。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SAND/WATR/GAS | 被完全阻挡 |
| PHOT/NEUT | 被完全阻挡 |
| 气压 | 自由穿过，两侧逐渐均衡 |
| SPRK | 不可传导 |
| 爆炸冲击 | 粒子被阻挡，但爆炸产生的气压波可以穿透 |

### 应用场景与构建示例

**场景 1：透气隔离容器**
```
         ═══[压力墙]═══（顶部透气）
        |               |
        |   [反应区]    |
        |               |
        ═══[基础墙]═════（底部封闭）
```
反应容器底部用基础墙（防止泄漏），顶部用压力墙。这样反应产生的气压可以释放到外部，防止内部压力积聚过高导致容器破裂或异常行为。同时反应物被完全困在内部。

**场景 2：气压传感器保护罩**
```
[外部环境] |═══[压力墙]═══| [气压传感器（TSNS/检测器+精密装置）]
```
在气压传感器前放置压力墙，让压力信号传入传感器但阻挡任何可能干扰传感元件的粒子（如腐蚀性气体、灰尘等）。

**场景 3：压力均衡的隔离舱**
```
[高压反应器 A] |═══[压力墙]═══| [隔离舱] |═══[压力墙]═══| [反应器 B]
```
两个独立的反应器通过压力墙连接到一个隔离舱。压力可以在三者之间均衡，但各自的内容物完全隔离——适合需要共享压力环境但避免交叉污染的多反应器系统。

**场景 4：声学/压力波导管**
多个压力墙排列成通道，引导气压波沿特定路径传播。虽然粒子无法通过，但压力脉冲（如爆炸产生的冲击波）可以沿压力墙通道传导到远处的检测器，实现"无线"气压信号传输。

冬季场景中，SNOW 和其他杂物混合在一起。将混合物倒在粉末墙上，SNOW（粉末类型）穿过墙体被收集，树叶（WOOD/PLNT）、石块（STNE 固体块）等被留在上方。

## 导体 (Conductor)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_CNDTR` |
| 颜色 | 淡黄色（半透明，几乎不可见） |
| 通透性 | 完全通透 — 所有粒子和压力均可自由穿过 |
| 导电性 | **是**（可在导体之间传导 SPRK 信号） |
| 特殊属性 | 隐形导线 — 对物质完全透明，仅作为电路通道存在 |

### 核心行为

导体是导体墙（ConductiveWall）的反向设计——它对物质完全透明，但保留导电性。它相当于一种"看不见的金属导线"。

- **完全透明**：所有类型的粒子（粉末、液体、气体、能量、固体）均可自由穿过，行为如同该格子为空。
- **压力通透**：气压完全不受阻碍。
- **导电性**：SPRK 信号可在相邻的导体格子之间传播，传播方式与普通金属导线（METL）一致。
- **无物理存在感**：粒子不会与导体碰撞、反弹或被加热（除非导体传导 SPRK 时附带的热效应）。导体对粒子运动轨迹无任何影响。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SAND/DUST/WATR/GAS | 完全自由穿过，不受任何影响 |
| PHOT/NEUT | 完全自由穿过 |
| SPRK | 沿导体传播，速度与 METL 导线相同 |
| 气压 | 完全自由穿过 |
| 相邻的 PSCN/NSCN | 可从导体接收 SPRK 或将 SPRK 传给导体 |
| INST | 可隔断导体电路 |

### 应用场景与构建示例

**场景 1：跨越粒子流的隐形电路**
```
[信号源] → ═══[导体]═══ → [目标元件]
               ↑
         （WATR 在此流过，
           不受导体影响）
```
在液体或气体管道中放置导体，构建穿过流体但不影响流体流动的电路。用于需要在粒子通道中传输电信号的场景——比如在水流中间设置控制点。

**场景 2：多层电路桥接**
```
[上层信号线（导体）] ──跨过── [中层粒子管道] ──到达── [下层信号线（导体）]
```
在复杂装置中，当普通导线（METL）会阻碍粒子流动时，用导体代替导线穿越粒子通道区域。导体不影响通道中的粒子，同时完成上下层电路的桥接。

**场景 3：分布式传感器网络**
在大型反应容器内部用导体构建一个网格状的信号网络。由于导体不阻挡任何粒子，网格的存在不会影响容器内的物理过程。但一旦任何位置的粒子触发了某个检测器，信号可以沿导体网络迅速传递到所有角落。

**场景 4：逻辑电路中的"跳线"**
```
[PSCN] → [导体（跳线，跨过其他线路）] → [NSCN] → [目标]
```
在密集的电路板布局中，使用导体作为"跳线"跨过下方的其他电路元件。由于导体不阻挡粒子，即使跳线下方有粒子流过，也不影响跳线的导电功能。

## 电锁体 (E-Hole)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_EHOLE` |
| 颜色 | 紫色 |
| 通透性 | 未通电时吸收一切（销毁）；通电时释放已吸收的粒子 |
| 导电性 | **是**（接收 SPRK 信号来控制释放） |
| 特殊属性 | 粒子暂存器 — 吸收粒子有记忆，通电后按吸收顺序释放 |

### 核心行为

电锁体是一种具有"记忆"功能的吸收墙。它在未通电时吸收并储存粒子，通电时将储存的粒子逐一释放出来。

- **未通电状态（吸收模式）**：所有接触到电锁体的粒子被吸收（从模拟中移除并存入内部缓存）。行为类似吸收墙，但粒子并未真正销毁——它们被暂存在电锁体的内部存储中。
- **通电状态（释放模式）**：当接收到 SPRK 信号时，电锁体开始释放之前吸收的粒子。释放按先进先出（FIFO）顺序进行——最先被吸收的粒子最先被释放。
- **存储容量**：每个电锁体格子有内部粒子缓存容量。当缓存满时，后续接触的粒子可能被丢弃（而非存储）。
- **压力行为**：与吸收墙不同，电锁体在未通电时也会阻挡气压（或气压行为取决于版本）。
- **释放的粒子**：释放出的粒子恢复其原有的温度、生命值（life）、ctype 等属性。

### 具体交互示例

| 粒子/元素 | 未通电 | 通电 |
|-----------|--------|------|
| SAND/DUST | 被吸收储存 | N/A（储存粒子被释放） |
| WATR/OIL | 被吸收储存 | N/A |
| GAS/SMKE | 被吸收储存 | N/A |
| PHOT/NEUT | 被吸收储存 | N/A |
| SPRK | 接收后触发释放模式 | N/A |
| 气压 | 可能被阻挡（取决于具体实现） | 取决于实现 |

### 应用场景与构建示例

**场景 1：粒子缓存器 — 脉冲式释放**
```
[持续粒子流] → ═══[E-Hole]═══ → [周期性释放到目标]
                    ↑
              [定时 SPRK 脉冲]
```
持续向 E-Hole 输入粒子（如 WATR），E-Hole 在未通电期间吸收并累积粒子。每隔一定时间发送一次 SPRK 脉冲，E-Hole 释放一批累积的粒子。实现"连续转脉冲"的粒子流量控制。

**场景 2：顺序记忆释放装置**
先后向 E-Hole 投入不同类型的粒子：先投入 SAND，再投入 WATR，再投入 GAS。通电后，E-Hole 按吸收顺序释放：先释放 SAND，再释放 WATR，再释放 GAS。利用这个 FIFO 特性可以实现基于时间顺序的粒子编排。

**场景 3：安全投料机构**
```
[危险物质储存] → → → ═══[E-Hole]═══ → [反应器]
                        ↑
                  [安全确认信号]
```
危险物质通过 E-Hole 门禁系统。只有安全确认信号（如温度达标、压力正常）到达后，E-Hole 才通电释放物质投入反应器。正常情况下物质被"扣押"在 E-Hole 中。

**场景 4：EMP 脉冲吸收 — 能量吸收器**
在设备周围布置 E-Hole，用于吸收突发的能量粒子爆发（如 PHOT 爆发）。未通电时 E-Hole 吸收所有能量粒子，保护内部设备。需要释放能量时通电，将能量粒子导入安全的排放通道。

**场景 3：气体传感器保护**

```
[外部环境] |═══[气体墙]═══| [GAS 检测器（检测器+信号处理）]
```
在气体检测器前放置气体墙，让目标气体（如 GAS、SMKE）进入检测器触发警报，但阻挡液体和粉末污染检测器。

**场景 4：蒸汽引擎管道**
```
[锅炉（WATR + HEAT）] → [WTRV 蒸汽] → ═══[气体墙管道]═══ → [涡轮/活塞端]
```
锅炉产生 WTRV 蒸汽，通过气体墙构建的管道输送到做功端。气体墙确保只有蒸汽通过管道，锅炉中的液态水被留在锅炉内继续加热。

## 引力墙 (GravityWall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_GRVTY` |
| 颜色 | 橙色 |
| 通透性 | N/A（特效型墙体，不阻挡任何物质） |
| 导电性 | 否 |
| 特殊属性 | 反重力场 — 范围内抵消牛顿引力（Newtonian Gravity），普通重力不受影响 |

### 核心行为

引力墙在 TPT 中是一个特殊效果墙体。它的核心功能是在其影响范围内取消牛顿引力（Newtonian Gravity）的作用。

- **对牛顿引力的影响**：牛顿引力是 TPT 中的一种引力模式（由 NEWT 元素或全局牛顿引力设置产生）。在这种模式下，所有粒子之间存在万有引力——质量越大的粒子相互吸引越强。引力墙在其所在区域内取消了这种引力效应。
- **对普通重力的影响**：引力墙不影响 TPT 的默认重力（即粒子向下掉落的普通重力）。粒子在该区域仍然会向下掉落。
- **范围**：引力墙的每一格都会取消该位置的牛顿引力。通常一排引力墙可以创建一个带状的无引力区。
- **不阻挡粒子**：引力墙本身不阻挡任何粒子，所有粒子可以自由穿过引力墙所在位置。粒子只是不受引力墙所在格子的牛顿引力影响。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| 所有粒子 | 自由穿过引力墙格子，不受阻碍 |
| NEWT（牛顿引力源） | 引力墙抵消 NEWT 产生的万有引力效应 |
| SAND/WATR/GAS | 仍然受普通重力影响向下运动，但在引力墙位置不受牛顿引力拉扯 |
| PHOT | 不受引力影响，引力墙对光子无效果 |
| SPRK | 不可传导 |

### 应用场景与构建示例

**场景 1：零重力实验区**
```
    ═══[引力墙]═══（天花板）
    |               |
    |  [零 G 区域]  |  ← 粒子在此区域内不受牛顿引力
    |               |
    ═══[引力墙]═══（地板）
```
在地板——而非天花板——放置引力墙，取消牛顿引力对区域内粒子的吸引力。用于研究粒子在无万有引力环境下的扩散和碰撞行为。

**场景 2：反引力悬浮装置**
```
    ═══[引力墙]═══  ← 位于悬浮目标下方
         ↑
    [悬浮的粒子云]
         ↑
    [NEWT 引力源]
```
在 NEWT 引力源上方放置引力墙，NEWT 试图将粒子向下吸引，但引力墙抵消了其引力作用，使粒子悬浮在半空。这是一种"反重力托盘"设计。

**场景 3：天体模拟 — 拉格朗日点**
在多体引力系统中放置引力墙，创建特殊的力平衡区域（模拟拉格朗日点）。粒子在这些区域内不受任何天体的引力影响，处于平衡状态。

**场景 4：引力隔离通道**
在充满 NEWT 引力的环境中，用引力墙构建一条"引力真空"通道。粒子在通道内不受外界大质量物体的引力干扰，可以沿直线运动。

## 阻压墙 (PressureWall)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_NOAIR` |
| 颜色 | 棕灰色 |
| 通透性 | 完全通透（物质侧）— 所有粒子均可穿过；完全阻挡（压力侧）— 气压无法穿过 |
| 导电性 | 否 |
| 特殊属性 | 透物不透气 — 与压力墙（NonpermeableWall）行为完全相反 |

### 核心行为

阻压墙是压力墙（NonpermeableWall）的逻辑反向。压力墙是"透气不透物"，阻压墙则是"透物不透气"。

- **粒子通透**：所有类型的粒子（粉末、液体、气体、能量、固体）均可自由穿过阻压墙，行为如同该格子不存在。
- **压力阻挡**：气压被完全阻挡。阻压墙两侧的压力完全隔离，一侧的高压不会扩散到另一侧。
- **不可导电**：不传导 SPRK 信号。
- **核心矛盾**：令人困惑的是——气体粒子（如 GAS、SMKE）可以穿过阻压墙，但气压不能穿过。这意味着单个气体分子可以穿过，但集体的压力波不能穿过。在实际模拟中，气体会在阻压墙附近产生有趣的堆积/稀疏行为。

### 具体交互示例

| 粒子/元素 | 交互行为 |
|-----------|----------|
| SAND/WATR/GAS | 全部自由穿过，不受阻碍 |
| PHOT/NEUT/ELEC | 全部自由穿过 |
| SMKE/WTRV | 自由穿过（但气压被阻挡可能导致这些粒子在墙体附近堆积） |
| 气压 | 被完全阻挡，两侧气压完全隔离 |
| SPRK | 不可传导 |
| PLSM | 自由穿过 |

### 应用场景与构建示例

**场景 1：压力隔离的粒子通道**
```
[高压区 A]  |═══[阻压墙]═══|  [低压区 B]
  WATR → → → → → → → → → → （WATR 穿过）
  压力不能穿过 → → → → → → （压力被隔离）
```
用于需要让粒子从高压区流向低压区（或反之），但不希望压力跟随传播的场景。阻压墙起到了"粒子单向阀 + 压力隔离器"的双重作用。

**场景 2：多压力区系统**
```
[区 1: 高压] |═══[阻压墙]═══| [区 2: 中压] |═══[阻压墙]═══| [区 3: 低压]
     粒子可自由在三区之间穿梭，但各区压力独立保持
```
构建多个相邻但压力独立的工作区。粒子（如 WATR、GAS）可以自由穿过各区域，但每个区域的压力由各自的内部源独立控制。适合模拟分层大气、多级压力系统等。

**场景 3：防压力回传**
```
[风扇] → [气流通道] → [目标区域]
                 |
           ═══[阻压墙]═══（侧面隔离）
                 |
           [旁路区域（不受气压影响）]
```
在气流通道侧面放置阻压墙，阻挡气压泄漏到旁路区域，防止旁路区域的粒子被意外吹动。但旁路区域的粒子可以自由穿越阻压墙进入主通道（如果需要的话）。

**场景 4：压力梯度控制系统**
```
[高压源] |═══[阻压墙 1]═══| [中压缓冲区] |═══[阻压墙 2]═══| [低压排放]
  粒子：自由穿过          粒子：自由穿过          粒子：自由穿过
  压力：到阻压墙 1 即停止  压力：独立              压力：独立
```
通过多个阻压墙将系统划分为多个压力独立区段。每个区段内的压力由各自的压力源控制，但粒子可以在整个系统中自由流动。可用于需要精确压力控制的多级反应系统。

## 清除一切 (Eraser)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_ERASEA` |
| 颜色 | 无（工具型，不渲染颜色） |
| 通透性 | N/A（工具而非墙体） |
| 导电性 | 否 |
| 特殊属性 | 全部清除 — 同时删除墙类、粒子和标记（Signs） |

### 核心行为

清除一切是一个强力编辑工具。它比清除墙（Erase）的功能更全面，一次操作可以同时清除三种不同类型的内容。

- **删除墙类**：移除被点击位置的所有墙体（与 Erase 相同）。
- **删除粒子**：移除被点击位置的所有粒子（粉末、液体、气体、能量、固体全部清除）。
- **删除标记**：移除被点击位置的标记（Signs）。TPT 中的标记是用于标注/保存文字信息的特殊元素。
- **作用范围**：仅影响被点击或拖拽经过的格子。

### 与清除墙 (Erase) 的对比

| 特性 | 清除墙 (Erase) | 清除一切 (Eraser) |
|------|---------------|-------------------|
| 删除墙 | 是 | 是 |
| 删除粒子 | 否 | 是 |
| 删除标记 (Signs) | 否 | 是 |
| 适用场景 | 仅修改墙体 | 完全清空区域 |

### 应用场景

- 快速重置整个实验区域
- 清理测试后残留的粒子和墙体
- 在复杂装置中清理特定区域以重新设计
- 删除意外放置的标记

## 静止墙 (Freezes)

| 属性 | 值 |
|------|-----|
| 内部标识 | `DEFAULT_WL_STASIS` |
| 颜色 | 青色（未通电时较亮；通电时变暗或颜色变化） |
| 通透性 | 物质可存在于格子内，但未通电时粒子被冻结无法移动 |
| 导电性 | **是**（接收 SPRK 信号来解除冻结状态） |
| 特殊属性 | 时间暂停 — 冻结范围内的所有粒子运动，通电时恢复 |

### 核心行为

静止墙是一种"时间冻结"型墙体。它不阻挡粒子进入其所在格子，但进入后的粒子在未通电状态下会完全丧失运动能力。

- **未通电状态（冻结模式）**：该格子内的所有粒子被冻结——它们不能移动、不能反应、不能进行热传导（取决于版本实现）。粒子仍然"存在"于格子中，但处于完全静止状态。检测器可以检测到这些被冻结的粒子。
- **通电状态（解冻模式）**：当接收到 SPRK 信号时，冻结效果解除，格子内的所有粒子恢复正常的物理行为——它们从被冻结的位置开始继续运动。
- **粒子进出**：粒子可以自由进入未被占满的静止墙格子（但进入后被冻结）。在通电状态下粒子可以自由进出。
- **压力行为**：气压在未通电时可能被阻挡或减缓，通电后恢复正常。

### 具体交互示例

| 粒子/元素 | 未通电 | 通电 |
|-----------|--------|------|
| SAND/DUST | 进入后被冻结，静止悬浮 | 恢复运动（重力下落） |
| WATR/OIL | 进入后被冻结，静止 | 恢复流动 |
| GAS/SMKE | 进入后被冻结，静止不扩散 | 恢复扩散 |
| PHOT/NEUT | 进入后被冻结（光子"停在原地"） | 恢复传播 |
| 检测器 | 可检测到被冻结的粒子，输出 SPRK | 正常检测 |
| SPRK | 接收到后进入解冻模式 | N/A |
| FIRE | 进入后被冻结，火焰暂停燃烧 | 恢复燃烧 |

### 应用场景与构建示例

**场景 1：粒子储存器 — 长期暂存**
```
[粒子入口] → → → ═══[静止墙区域]═══ → [粒子出口（E-Wall）]
                     （粒子被冻结保存）
```
将需要长期储存的粒子导入静止墙区域。粒子进入后被冻结，不会反应、不会消散、不会变质。需要使用时打开出口的 E-Wall，同时向静止墙通电解冻，粒子恢复运动并从出口流出。

**场景 2：时序控制 — 延迟启动**
```
[触发器] → [SPRK] → [经过延迟后到达静止墙] → [解冻关键粒子] → [反应开始]
```
某个关键反应物被冻结在静止墙中。只有当外部条件满足（如温度达标、前级反应完成）后，控制电路才向静止墙发送 SPRK 信号，解冻反应物，反应才开始进行。这实现了精确的时序控制。

**场景 3：防爆安全锁**
```
[爆炸物（如 DEUT/PLUT）] → ═══[静止墙]═══ → [引爆区域]
                              ↑
                        [安全确认信号]
```
爆炸性物质先经过静止墙区域，被冻结暂存。只有在安全确认信号到达（解冻）后，爆炸物才继续前进到达引爆区域。如果安全条件未满足，爆炸物被永久冻结在原位，不会意外爆炸。

**场景 4：帧定格观察窗**
```
[运动中的高速粒子] → ═══[静止墙观察窗]═══ → [继续运动]
                        （解冻后观察）
```
在粒子运动路径上放置一排静止墙，粒子进入后被冻结。此时可以仔细观察粒子的温度、ctype、life 等属性（通过暂停模拟查看）。通电解冻后粒子继续运动，不妨碍后续实验。这是一种"非破坏性快照"工具。

**场景 5：粒子队列 — FIFO 缓冲区**
```
[粒子流] → ═══[静止墙 1]═══ → ═══[静止墙 2]═══ → ═══[静止墙 3]═══ → [输出]
```
将静止墙排成一列，构建粒子队列。通过依次通电解冻各段静止墙，可以实现粒子的精确顺序释放，配合其他电路元件实现复杂的粒子编排逻辑。
