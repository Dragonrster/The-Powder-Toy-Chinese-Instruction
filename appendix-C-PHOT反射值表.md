# 附录C: PHOT元素反射值表

`PhotonReflectWavelengths` 是一个 32 位位掩码，每一位代表一个特定波长的光子（PHOT）是否会被该元素反射。

## 工作原理

- 光子的 Ctype 字段存储其波长编码（低 26 位，每 2 位编码一个颜色分量）
- 当光子穿过某元素粒子时，系统检查该元素的 PhotonReflectWavelengths 是否包含光子当前波长
- 若匹配：光子被**反射**（随机改变方向，不穿过粒子）
- 若不匹配：光子**穿透**粒子继续传播（前提是该元素有 PROP_PHOTPASS 属性）
- 掩码为 `0x00000000` 表示不反射任何波长——所有光子可穿透
- 掩码高位(bit31~26)通常为 0

## 常见 PHOT 反射关系

| 元素 | 光子行为 |
|------|---------|
| FILT | 不反射但**修改**光子波长（通过 tmp 模式控制） |
| QRTZ/PQRT | 不反射但**散射**光子（随机改变方向和波长） |
| BGLA | 不反射但**折射**光子（随机旋转方向） |
| INSL | 不反射，光子可穿透(PROP_PHOTPASS) |
| GLAS | 不反射，光子穿过时增强光线效果 |
| LCRY(激活态) | 不反射，光子透过时改变颜色 |

## 掩码位映射参考

| 位范围 | 颜色通道 | 说明 |
|--------|---------|------|
| bit0~9 | 红色分量 | 每 2 位编码 1 级（共 5 级） |
| bit10~19 | 绿色分量 | 同上 |
| bit20~26 | 蓝色分量 | 每 2 位编码 1 级（共 3.5 级） |

实际编码规则：波长 = (蓝分量 << 20) | (绿分量 << 10) | 红分量。每个颜色分量 10 位空间中每 2 位表示一个亮度级别。
