# 附录I：PIPE/PPIP管道系统详解

## 管道基本原理

管道使用**三色循环系统**实现单向运输：

```
RED(红) → BLUE(蓝) → GREEN(绿) → RED(红)
```

粒子只能沿颜色方向流动，不会倒流。

---

## 管道创建过程（分步详解）

### 步骤1：放置管道粒子
放置PIPE或PPIP粒子。推荐使用画刷(中/大)批量放置，确保粒子连续。

### 步骤2：初始化（life>10阶段）
粒子放置后，life随时间增长。当life>10时：
- 自动检测周围**最多2格**范围内的空间
- 在**5x5区域**内创建BRCK砖块边框
- BRCK形成管道的物理外壳
- 同一管段内会自动检测是否有冲突方向

### 步骤3：颜色分配（life==3阶段）
当life恰好为3时（初始化后期），相邻的初始化中PIPE之间：
- 按**RED → BLUE → GREEN → RED**循环分配颜色
- 颜色沿管道延伸方向依次循环
- 形成完整的封闭管道网后停止分配

### 步骤4：管道就绪
整个管段形成统一的方向标记后，管道进入正常工作状态。

## 管道颜色系统

### 颜色编码（通过Ctype存储）

| 颜色 | Ctype值 | 流向 | 示意图 |
|------|---------|------|--------|
| RED (红) | 1 | 红色段→蓝色段 | R→B→G→R |
| BLUE (蓝) | 2 | 蓝色段→绿色段 | B→G→R→B |
| GREEN (绿) | 3 | 绿色段→红色段 | G→R→B→G |
| 无色 (入口) | 无标志 | 吸收粒子进入管道 | 入口态 |

### 颜色流向判断逻辑

```
当前段颜色 → 下一个目标颜色
---------------------------
RED    → 寻找BLUE段
BLUE   → 寻找GREEN段
GREEN  → 寻找RED段
入口    → 吸收→分配→进入管道
```

---

## 粒子运输流程（逐帧详解）

### 第1帧：入口吸收
1. 管道口（无colour标志的PIPE）扫描**3x3范围**
2. 检测接触到的粒子
3. 将粒子"吸入"管道内部存储
4. 粒子的所有属性被保存（见下）

### 第2帧：管道推进
1. `pushParticle`函数在8邻域中搜索
2. 查找颜色为`nextColor`的空PIPE格
3. 找到后立即将粒子移至该格
4. 递归调用，每帧最多推进**2格**

### 第3帧：出口释放
1. 当粒子到达管道末端（无下一个颜色段）
2. 在末端相邻空位执行`create_part`释放粒子
3. 粒子恢复所有保存的原始属性

---

## 存储的粒子属性

管道中运输的粒子完整保留：

| 存储字段 | 原始属性 | 说明 |
|---------|---------|------|
| Ctype | 粒子类型 | 运输粒子的类型ID |
| Temp | 温度 | 保持原始温度不变 |
| Dcolour | 装饰色 | 粒子外观颜色 |
| Tmp2 | life值 | 原始生命值 |
| Tmp3 | tmp值 | 原始临时数据 |
| Tmp4 | ctype值 | 原始携带类型 |

## PPIP可控管道

与PIPE共用核心逻辑，额外支持电信号控制：

### 电信号控制表

| 信号元素 | 触发方式 | 效果 | Tmp标志位 |
|---------|---------|------|-----------|
| PSCN | Flood_trigger | 清除PAUSED标志，开启运输 | 0x10000000 |
| NSCN | Flood_trigger | 设置PAUSED标志，停止运输 | 0x08000000 |
| INST | Flood_trigger | 反转运输方向(REVERSED) | 0x04000000 |

### Tmp标志位完整系统

| 位 | 标志 | 十六进制 | 说明 |
|----|------|---------|------|
| 0 | HEAT_MODE | 0x00000001 | 导热模式（检测HEAC激活） |
| 24 | DIRECTION_REVERSED | 0x01000000 | 方向已反转 |
| 25 | PAUSED | 0x02000000 | 暂停运输 |
| 26 | INST_RECEIVED | 0x04000000 | 收到INST反转信号 |
| 27 | NSCN_RECEIVED | 0x08000000 | 收到NSCN关闭信号 |
| 28 | PSCN_RECEIVED | 0x10000000 | 收到PSCN激活信号 |
| 29-31 | 保留 | — | 未使用 |

### PPIP状态转换图

```
         PSCN激活
PAUSED ──────────▶ ACTIVE
  ▲                   │
  │    NSCN关闭       │ INST反转
  └───────────────────┘
                      │
                      ▼
                  REVERSED
                  (方向反转)
```

## 管道互操作

### 管道↔STOR（存储容器）
```
PIPE ──提取──▶ STOR
STOR ──注入──▶ PIPE
```
- 管道从STOR提取粒子时保留STOR内已有粒子的Ctype和Temp
- 管道向STOR注入时粒子存储在STOR内部结构中

### 管道↔PRTI（传送门入口）
```
PIPE端 ──转运──▶ PRTI ──传送──▶ PRTO ──释放
```
- 管道末端粒子转入传送门入口
- 粒子通过传送门瞬间到达PRTO位置
- **显著加速**运输速度（跳过中间管道段）

### 管道↔BRCK（砖块）
- PAUSED状态变化时控制BRCK发光
- `tmp=1`: BRCK发光（管道激活）
- `tmp=0`: BRCK不发光（管道暂停）
- 可用于**视觉信号指示**

## 单像素管道

Tmp中有**0x100**标志时启用单像素模式。

### 方向位编码

| 方向 | bit10-12 (前向) | bit13-14 (反向) | 说明 |
|------|----------------|----------------|------|
| 右 | 0x400 | — | 向右推进 |
| 左 | — | — | 向左推进 |
| 上 | — | — | 向上推进 |
| 下 | — | — | 向下推进 |

单像素管道沿**预设方向**推送，比随机搜索更快。

## 管道系统限制

| 限制项 | PIPE | PPIP |
|--------|------|------|
| 高压破坏(>10.0) | 是 → BRMT | 否（抗压） |
| 电信号控制 | 否 | 是 |
| 最大运输速度 | 2格/帧 | 2格/帧 |
| 封闭循环要求 | 是 | 是 |
| 导热模式 | 否 | 是(标志位控制) |

## 分步搭建指南

### 基础管道回路搭建

```
步骤1: 创建封闭管道环
─────────────────────────
用矩形刷放置PIPE形成闭合回路，例如：
    ┌─────────────┐
    │             │
    │  闭合回路    │
    │             │
    └─────────────┘

步骤2: 留出入口/出口
──────────────────────
在一侧留出不闭合的PIPE段作为入口：
    入口→ ┌─────────┐
          │ 封闭区域 │
          └─────────┘→ 出口

步骤3: 等待初始化
──────────────────
管道自动初始化，分配颜色。观察颜色是否正确循环。

步骤4: 测试运输
────────────────
在入口处放置WATR粒子，观察是否沿管道流动到出口。
```

### 常见配置1：单向传输线
```
入口[PIPE无颜色] → [RED] → [BLUE] → [GREEN] → [RED] → ... → 出口[PIPE无颜色]

用途：将粒子从A点运输到B点
```

### 常见配置2：双向管道（使用PPIP）
```
         ┌──── RED ──▶ BLUE ──▶ GREEN ──┐
入口1 ──▶│                               │──▶ 出口1
         │    PPIP + INST信号反转         │
入口2 ◀──│                               │◀── 出口2
         └──── RED ◀── BLUE ◀── GREEN ──┘
```

### 常见配置3：管道分流器
```
              ┌──▶ 分支1出口
入口 ──▶───┤
              └──▶ 分支2出口

使用STOR + 多个管道出口实现选择性分流
```

### 常见配置4：管道+传送门加速
```
[长距离PIPE管道]──▶ [PRTI] ～～传送～～ [PRTO]──▶ [继续PIPE管道]
                           大幅缩减传输时间
```

## 故障排除指南

### 问题1：粒子不进入管道
**可能原因**：
- 管道未完成初始化（life还不够大）
- 管道未形成封闭回路
- 入口段没有颜色标志
- 粒子被WALL阻挡无法靠近

**解决方法**：
- 等待管道life增长到>10
- 确保管道回路完整闭合
- 检查入口段PIPE是否在3x3范围内
- 清除入口附近的障碍物

### 问题2：粒子卡在管道中不动
**可能原因**：
- 颜色循环断裂（中间缺了一段）
- PPIP收到了PAUSED信号
- 管道高压(>10)已破裂成BRMT
- 出口被堵住

**解决方法**：
- 检查管道颜色是否完整连续（R→B→G→R→...）
- 移除NSCN信号，发送PSCN激活
- 重建破裂的管段
- 确保出口有至少1格空位

### 问题3：粒子方向错误（倒流）
**可能原因**：
- 颜色分配环混乱（两条管道相交）
- PPIP的INST信号方向反转

**解决方法**：
- 避免管道交叉，保持回路独立
- 检查INST信号连接

### 问题4：运输速度太慢
**可能原因**：
- 管道过长
- 粒子量大，排队

**解决方法**：
- 使用PRTI/PRTO传送门跳过中间段
- 使用单像素管道(Tmp 0x100)加速
- 多开并行管道

## 高级管道技术

### 粒子过滤器管道
结合PPIP的PAUSED控制和检测逻辑：
```
[入口]→[检测段(PSCN触发)]→条件通过→[PPIP ACTIVE]→[出口]
                         →条件不通过→[PPIP PAUSED]→粒子暂存
```
实现仅运输特定类型或满足条件粒子的智能管道。

### 定时运输管道
```
PSCN脉冲发生器 → PPIP管道
        产生周期性信号
        管道按周期开启/关闭
```

### 管道网络拓扑
```
         ┌─ 管道A ─┐
入口 ──▶┤ 管道B ├──▶ STOR ──▶ 出口
         └─ 管道C ─┘
```
多路管道并行输入到STOR，再统一输出。
