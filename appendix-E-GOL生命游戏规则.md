## 各类规则详细说明

### 0. GOL (Conway's Game of Life) — 经典基准

- **B3/S23**: 3个邻居出生，2或3个邻居存活
- **著名模式**：
  - **静止体**：Block(2x2方块)、Beehive(蜂巢)、Loaf(面包)、Boat(船)、Tub(浴缸)
  - **振荡子**：Blinker(周期2,3格线)、Toad(周期2)、Beacon(周期2)、Pulsar(周期3)
  - **飞船**：Glider(滑翔机,周期4逃逸)、LWSS(轻型太空船)、MWSS/HWSS(中/重型)
  - **繁殖者**：Gosper Glider Gun(滑翔机枪,周期30)、Puffer系列
- **实用场景**：作为其他规则的"基准参考"，常与自定义规则对比

### 1. HLIF (High Life) — 复制大师

- **B36/S23**: 比GOL多B6
- **关键特征**：B6条件使6个邻居围绕空格的图案可产生新细胞
- **著名模式**：Replicator(自复制体,每12代产生一个副本并平移)
- **实用场景**：自动填充、周期性图案生成器

### 2. ASIM (Assimilation) — 钻石扩张

- **B345/S4567**: 高生存条件使图案高度稳定
- **关键特征**：一旦成型几乎不可摧毁，呈现菱形边界扩张
- **实用场景**：稳定防护层、缓慢但确定的区域填充

### 3. 2X2 — 最小方块

- **B36/S125**: 所有稳定结构以2x2块为基本单位
- **关键特征**：不会产生大于2x2的静止体
- **实用场景**：高密度存储、2x2块纹理

### 4. DANI (Day and Night) — 反转对称

- **B3678/S34678**: 将图案反色(B/W互换)后行为完全相同
- **关键特征**：斑马条纹图案，日夜模式对称演化
- **实用场景**：美学图案、对称性研究

### 5. AMOE (Ameba) — 变形虫

- **B357/S1358**: 产生大块不规则团状结构
- **关键特征**：边界模糊，像阿米巴变形虫一样缓慢扩张
- **实用场景**：有机形态生成、慢速外扩

### 6. MOVE — 伪移动

- **B368/S245**: 产生斜向传播的图案，看起来像对角线移动
- **关键特征**：实际是死亡-新生交替，并非真正移动
- **实用场景**：方向性纹理生成

### 7. PGOL (Pseudo Life) — 混沌伪生命

- **B357/S238**: 与GOL相比S2/S3互换、B5替代B2
- **关键特征**：行为介于有序和混沌之间，不可预测
- **实用场景**：混沌行为研究

### 8. DMOE (Diamoeba) — 钻石变形虫

- **B35678/S5678**: 极高生存率，大部分存活细胞不死
- **关键特征**：形成大块钻石形固体
- **实用场景**：大面积快速填充

### 9. 3-4 — 极简晶体

- **B34/S34**: 出生和存活规则相同
- **关键特征**：产生稳定的晶体结构，极简但高度有序
- **实用场景**：晶体生长模拟

### 10. LLIF (Long Life) — 长寿振荡

- **B345/S5**: 大多数新图案在极长时间后才稳定
- **关键特征**：振荡周期极长，有些图案需数万代才稳定
- **实用场景**：长时间演化观察

### 11. STAN (Stains) — 快速填充

- **B3678/S235678**: 极多的出生和存活条件
- **关键特征**：像墨水污渍一样快速填充空间
- **实用场景**：快速区域填充、背景生成

### 12. SEED — 注定消亡

- **B2/S**: 无存活条件(空S)，所有图案终将死亡
- **关键特征**：从2个邻居产生新细胞后立即在下代死亡，但爆炸式扩张
- **实用场景**：瞬态装饰效果、临时图案

### 13. MAZE — 迷宫生成器

- **B3/S12345**: 广泛存活条件+严格出生条件
- **关键特征**：产生细长连接的迷宫状通道
- **实用场景**：迷宫生成、迷宫解算演示

### 14. COAG (Coagulations) — 交替凝固

- **B378/S235678**: 生长-死亡周期性交替
- **关键特征**：凝固边缘向外扩张，内部周期性空洞
- **实用场景**：多孔结构生成

### 15. WALL (Walled Cities) — 城墙城市

- **B45678/S2345**: 高出生需要+中等存活
- **关键特征**：形成厚壁围成的独立"城市"区域
- **实用场景**：分区填充、房间结构生成

### 16. GNAR — 最小规则

- **B1/S1**: 仅需1个邻居，规则最小化极限
- **关键特征**：任何单格接触即触发，缓慢向外膨胀
- **实用场景**：膨胀行为演示

### 17. REPL (Replicator) — 完美复制

- **B1357/S1357**: 所有奇数邻居触发
- **关键特征**：图案产生自身的完美复制品（时间和空间平移）
- **实用场景**：自复制系统研究

### 18. MYST (Mystery) — 神秘规则

- **B3458/S05678**: 0邻居存活使孤立的活细胞永不死
- **关键特征**：行为难以预测，产生各种意外形态
- **实用场景**：探索性研究

## 多态规则状态机（文本描述）

### 3态规则 (FRG2/FROG/BRAN)

```
状态0(死亡) ──出生──▶ 状态1(新生,亮色) ──存活──▶ 状态2(成年,暗色)
    ▲                                                    │
    └────────────────── 死亡 ─────────────────────────────┘
```

- **状态0**: 死亡/空，不参与任何计算
- **状态1**: 新生细胞，高亮显示，参与邻居计算(计为存活)
- **状态2**: 成年细胞，暗色显示，参与邻居计算(计为存活)
- **颜色作用**：产生自然的渐变效果，新生区域亮、成熟区域暗

### 4态规则 (LOTE)

```
状态0 ──B──▶ 状态1 ──S──▶ 状态2 ──S──▶ 状态3 ──S──▶ 状态0
(红)         (橙)         (黄)         (金色)
```

- B37/S3458，但需连续3帧存活才稳定
- 颜色随状态从红色渐变至黄色
- 产生闪烁/流动视觉效果

### 6态规则 (STAR)

```
状态0 ──B──▶ 状态1 ──▶ 状态2 ──▶ 状态3 ──▶ 状态4 ──▶ 状态5
(深蓝)       (蓝)     (浅蓝)     (青色)    (白)      (最亮)
```

- B278/S3456，6个生命状态渐变
- 颜色从深蓝渐变至亮白
- 模拟Star Wars规则中的多种生命形态

## 多GOL类型交互

多个不同规则的生命形式可以共存于同一空间：

### 交互规则
- **隔离开关**：不同Ctype的LIFE粒子**相互视为死亡(0)细胞**——即GOL(0)和HLIF(1)互相不可见
- **共存方式**：可在同一画布放置多种LIFE类型，各自独立演化
- **边界效应**：两种LIFE类型接触边界处互相不计算邻居，形成自然隔离

### 实际应用示例

```
┌───────────────────────┐
│ GOL区              │  GOL区    │  GOL区    │
│ (经典滑翔机枪)      │  ←不交互→ │  ←不交互→ │
│                     │           │           │
├───────────────────────┤
│ HLIF区              │
│ (自复制模式)         │
└───────────────────────┘
```

### 混合实验建议
- **进化竞赛**：同时运行GOL和HLIF，对比演化速度
- **规则分层**：WALL规则作为"墙壁"层，其他规则作为"生物"层
- **信号转换**：利用Lua脚本读取GOL区状态，转换为FILT/电信号

## 自定义GOL规则

### 通过Lua添加

```lua
-- 添加自定义规则
sim.addCustomGol("MyRule", 3, 4, 5,  2, 3)  -- B345/S23
-- 语法: sim.addCustomGol(name, b1, b2, b3, ..., s1, s2, ...)

-- 列出所有规则
local rules = sim.listDefaultGol()
for i, r in ipairs(rules) do
    tpt.log(string.format("%d: %s (B%s/S%s)", i-1, r.name, r.borns, r.survives))
end

-- 设置LIFE粒子规则
local golParticle = sim.partCreate(-1, 100, 100, "LIFE")
if golParticle > 0 then
    sim.partProperty(golParticle, 2, 1)  -- ctype=1 → HLIF
end

-- 批量设置规则
for i in sim.parts() do
    if sim.partProperty(i, 0) == elements.getByName("LIFE") then
        sim.partProperty(i, 2, 5)  -- 全部设为AMOE
    end
end
```

### 删除自定义规则
```lua
sim.removeCustomGol("MyRule")  -- 删除指定的自定义规则
```

## 明暗/颜色效果

所有规则的颜色可以通过温度或Lua脚本动态变化，产生丰富的视觉效果：

- **温度渐变**：高温LIFE粒子颜色偏向暖色，低温偏向冷色
- **Dcolour覆盖**：通过Lua设置`partProperty(i, 13, 0xRRGGBB)`自定义颜色
- **LIFE→DEAD转换**：LIFE在过高温(>9000K)时转化为能量类型

## 使用方式

在LIFE粒子的Ctype中设置规则编号(0~23)即可切换。可通过Lua的`sim.listDefaultGol()`遍历，`sim.addCustomGol()`添加自定义规则。

### 小贴士
- 使用**保存印章**(Save Stamp)功能保存喜欢的GOL图案，方便重复使用
- 结合**PPIP导电管道**可实现根据电信号切换GOL规则
- 配合**FILT滤镜**可将不同GOL区域产生的颜色转换为不同波长光子
- 参考附录D了解完整的Lua API操作LIFE粒子
- 参考附录P了解粒子标志位对GOL渲染的影响
