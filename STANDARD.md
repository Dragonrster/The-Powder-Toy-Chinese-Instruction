# TPT 元素文档撰写标准

基于 cray.md 的文档结构提取。每个元素的文档应至少覆盖以下章节。

---

## 标准结构

### 1. 属性表（必选）
每个元素开头应有简洁的属性表，包含内部标识、颜色、元素类型、重量、硬度、热导率、菜单分类等基本信息。格式统一使用 Markdown 表格。

**必填字段**：
- 内部标识（DEFAULT_PT_XXX）
- 颜色（十六进制 RGB + 中文颜色名）
- 元素类型（固体/液体/气体/粉末/能量 + TYPE_XXX）
- 重量
- 硬度
- 热导率

**选填字段**：特殊属性标志位、CarriesTypeIn、默认温度、状态转换阈值等。

### 2. 描述与核心机制（必选）
- 一句话描述元素的核心功能
- 主要行为逻辑（Update/Graphics 函数的核心流程）
- 关键的工作阶段或状态
- 与其他元素的根本区别

### 3. 参数控制系统（如有）
- 各参数（tmp/tmp2/life/ctype/temp 等）的含义和作用
- 默认值和取值范围
- 参数之间的交互关系
- 如何通过外部工具（HEAT/COOL/PROP）调节参数

### 4. 特殊交互机制（必选）
- 与其他元素的特殊交互
- 触发条件和结果
- 颜色/温度/状态传递规则
- 对极端环境（高温/高压/真空）的响应

### 5. 粒子创建与属性继承（如有）
- 创建粒子的条件和过程
- 属性（温度、生命、颜色、类型）的继承规则
- 创建失败/被阻止时的回退行为

### 6. 墙体/环境交互（如有）
- 与各类型墙体的交互（阻挡/穿透/触发）
- 与压力场/温度场的互动
- 重力、空气阻力等环境因素的影响

### 7. 应用场景（必选）
- 至少给出 2-3 个具体使用场景
- 与其他元素的组合使用方法
- 在知名存档/机器中的典型应用

### 8. 反应链（如有）
- 元素参与的所有化学反应/物理变化
- 使用统一格式：`A + B → C (概率, 条件)`

### 9. 调试信息（如有）
- 调试模式(D键)下显示的额外信息
- 对构建者有用的内部状态指示

---

## 撰写原则

1. **以源码为准**：所有行为描述必须有源码依据，不猜测。数据来自 `elements_data.json`（由 `extract_elements.py` 从 C++ 源码提取）
2. **中文优先**：元素名保留英文 4 字母代码（如 METL、SPRK），其余全部用中文
3. **逐元素深入**：每个元素独立成节，不合并简述。即使简单元素也要写出与其他元素的区别
4. **代码注释式**：关键机制需描述到代码逻辑级别——"如何做"而不只是"做什么"
5. **实用导向**：每个元素至少给出一个具体使用场景（不是"可用于实验"这种空泛描述）
6. **数据精确**：温度用 `xxx℃/xxxK` 双单位，压力用数值+单位，概率用分数（1/100 而非 1%）
7. **交叉引用**：涉及其他元素时使用内部链接或明确写出元素名（英文代码+中文名）

## 文件头规范

每个分类文件应包含：
```markdown
# [EnglishName][中文名]

[一句话分类描述]。本分类共 **N** 个元素。

- [元素索引列表，每行格式：ELEM Type:ID — 简短描述]
```

## 元素节规范

```markdown
###### 中文名(CODE)【English Name】----------------------------------------------------Type:XXX

[属性表(如有)]

**描述：**...

### 核心机制(如有)
...

### 参数系统(如有)
...

**反应：**(如有)
- A + B → C
```

## 质量控制检查清单

- [ ] 属性表完整且格式统一
- [ ] 核心机制有源码依据
- [ ] 参数含义明确（不说"tmp用途不明"）
- [ ] 至少 2 个应用场景
- [ ] 温度使用 K/℃ 双单位
- [ ] 概率使用分数格式
- [ ] 反应链使用统一格式
- [ ] 交叉引用准确
- [ ] 无猜测内容（所有描述有据可查）
- [ ] 与 cray.md 参考深度一致或更深

## cray.md 参考

[cray.md](../cray.md) 是本项目的文档深度参考标准。每个元素文档应达到或超过其详细程度：

| cray.md 章节 | 对应标准 |
|-------------|---------|
| 基本属性 | 属性表 + 描述 |
| 核心工作机制 | 核心机制 |
| 参数控制系统 | 参数系统 |
| 特殊交互机制 | 特殊交互 |
| 粒子创建与属性继承 | 粒子创建 |
| 墙体阻挡检测 | 墙体交互 |
| 调试信息显示 | 调试信息 |
| 应用场景与设计理念 | 应用场景 |

**关键衡量标准**：如果 cray.md 对 CRAY 元素的描述能让读者完全理解其行为并独立使用，那你的文档也应达到同样水平。
