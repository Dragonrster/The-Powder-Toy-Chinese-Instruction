# Life生命类

基于Conway生命游戏规则（Game of Life），使用细胞自动机在摩尔邻域（Moore neighborhood）中产生各种图案和行为。

本分类共 **1** 个主元素（LIFE），通过 Ctype 切换 **24 种**内置规则。

---

## 核心概念：GOL规则语法

所有 GOL 元素遵循统一规则格式：

> **`B{数字}/S{数字}[/{态数}]`**

| 部分 | 含义 |
|------|------|
| **B (Begin/Born)** | 死亡细胞在周围存活邻居数匹配 B 中任一数字时诞生 |
| **S (Stay/Survive)** | 存活细胞在周围存活邻居数匹配 S 中任一数字时继续存活，否则死亡 |
| **态数 (States)** | 可选，默认为2。≥3时细胞经过中间"濒死"状态才完全死亡，产生颜色渐变的视觉效果 |

**经典示例** — Conway生命游戏 `B3/S23`：
- B3：空格子周围恰好3个活细胞 → 新生
- S23：活细胞周围2或3个活细胞 → 存活；其余情况（0-1个孤立，4-8个过度拥挤）→ 死亡

---

## 多规则共存机制

当不同 GOL 类型在同一邻域内共存时：

1. 所有类型的诞生规则（B）对邻域内**所有**存活细胞（不分类型）统一计算
2. 存活细胞数量**过半的类型**获得该细胞的"所有权"
3. 平局时，**内部编号较小**的类型获胜（对于内置规则，偏向菜单中靠右的类型）

这意味着你可以让不同规则的生命"竞争"同一片区域。

### 核心工作机制

LIFE 元素本身不运动、不下坠，完全按照细胞自动机规则演化。每一帧：

1. **邻居统计**：计算周围 8 个摩尔邻居中存活 LIFE 细胞的数量
2. **规则匹配**：根据 Ctype 指定的规则集判断"生"还是"死"
3. **状态更新**：创建新细胞或让细胞进入下一衰减状态

LIFE 的扩散范围受限于 `AirLoss = 0.90` 和 `Loss = 0.00`——空气阻力高，自身无速度损失，意味着它不会被风吹走但也不主动扩散。

### 参数系统

| 参数 | 用途 |
|------|------|
| **Ctype** | GOL 规则编号（0~23 选择内置规则）。也支持自定义规则字符串（通过 Lua `sim.addCustomGol()`） |
| **Dcolour** | 自定义颜色1（ARGB），设为0时使用规则默认颜色 |
| **Tmp** | 颜色2（用于多态规则的中间态），或存储自定义颜色编码 |
| **Temp** | 默认 9000K（极高），但生命类不受温度影响——无任何温度/压力状态转换 |
| **Life** | 当前生命状态计数，多态规则中表示当前所处的状态阶段 |

### 渲染与颜色

LIFE 元素的渲染使用双色系统：
- **Colour1（Dcolour）**：存活态颜色。为 0 时使用规则内置颜色
- **Colour2（Tmp）**：濒死/中间态颜色

对于多态规则（3+态）：
- 2态规则：存活=Colour1，死亡=消失
- 3态规则：存活=Colour1，濒死=Colour2，死亡=消失
- 4~6态规则：存活→渐变→Colour2→渐变→死亡消失

装饰层开启时，规则内置颜色优先；装饰层关闭时使用 Dcolour 和 Tmp 自定义色。

### 无状态转换

LIFE 不响应任何温度、压力触发——`LowPressure/IPL`、`HighPressure/IPH`、`LowTemperature/ITL`、`HighTemperature/ITH` 均为不转换（NT）。因此 LIFE 图案可在极端环境中稳定存在，不受外界物理条件干扰。

### 多态规则的颜色渐变

多态规则（态数≥3）中，细胞经历"存活→濒死1→濒死2→...→死亡"过程，每个状态持续1帧：

- **3态** (FRG2/FROG/BRAN)：存活(亮色) → 濒死(暗色) → 死亡
- **4态** (LOTE)：存活 → 濒死1 → 濒死2 → 死亡（闪烁感）
- **6态** (STAR)：存活 → 4个濒死态渐变 → 死亡（丰富的颜色层次）

## 自定义 GOL 规则

### 通过 Lua 添加自定义规则

```lua
-- 添加自定义规则
local ruleID = sim.addCustomGol(
    "B3/S23/3",    -- 规则字符串
    0xFF0000,       -- 存活颜色 (红)
    0x880000,       -- 中间态颜色 (暗红)
    "我的规则"       -- 名称
)

-- 将 LIFE 粒子设置为该自定义规则
-- ruleID 从 NGOL (24) 开始递增
sim.partProperty(partID, "ctype", ruleID)

-- 列出所有可用规则（内置+自定义）
local allRules = sim.listCustomGol()
for i, r in ipairs(allRules) do
    print(r.name, r.rule, r.color1, r.color2)
end

-- 删除自定义规则
sim.removeCustomGol(ruleID)
```

### 规则字符串格式

自定义规则使用与内置规则相同的 `B{}/S{}/{}` 格式：
- B 和 S 各可包含 0~8 的数字
- 态数（第三部分）可选，但**必须 ≥ 3**，不能为 1；2 态时省略
- 示例：`B278/S3456/6`（Star Wars 规则）

## 应用场景

1. **细胞自动机展示**：在存档中运行 Conway 生命游戏，展示涌现行为
2. **图案生成器**：利用不同规则产生独特的视觉纹理（MAZE 迷宫、WALL 城墙）
3. **逻辑电路**：GOL 本身是图灵完备的，可构建逻辑门和计算器
4. **自复制实验**：利用 REPL 规则观察精确自复制
5. **规则竞赛**：在同一区域放置不同规则的生命，观察竞争和共存
6. **装饰与艺术**：利用多态规则的渐变颜色创造视觉作品
7. **温度无关稳定性**：LIFE 不受温度/压力影响，可在任何环境中永久保存图案
